RISC-V Processor / HaDes-V Lab

SystemVerilog hardware-design project focused on implementing a modular, pipelined 32-bit RISC-V processor, including fetch, decode, execute, memory, writeback, register file, instruction decoder, CSR handling, interrupts, and Wishbone-based memory/peripheral access.

Overview

HaDes-V is a hardware-design laboratory project in which a RISC-V-based microcontroller is implemented in SystemVerilog. The course focuses on understanding the internal structure and operation of an MCU by designing hardware modules, writing low-level software, and debugging both hardware and software behavior.

The processor developed in the lab is a 32-bit pipelined RISC-V processor with five main stages: instruction fetch, instruction decode, execute, memory, and writeback. The design supports RV32I instructions, CSR instructions, machine-mode execution, timer behavior, and interrupt handling.

Technical Details

Pipeline Architecture

The processor is organized as a five-stage pipeline with fetch, decode, execute, memory, and writeback stages. Each stage passes instruction, program-counter, data, and status information forward, while control and jump-related signals can flow backward to handle stalls, jumps, traps, and interrupts.

Instruction Decoder and Register File

The decode logic translates raw RISC-V instructions into the control signals required by the later pipeline stages. The register file provides read and write access to the general-purpose integer registers and is a core part of instruction execution and forwarding behavior.

Execute Stage

The execute stage performs arithmetic, logical, branch, jump, and address calculation operations. It also contributes to the control-flow behavior of the processor by generating next-PC and jump-related information.

Memory Stage and Wishbone

The memory stage handles load and store operations and communicates with memory and peripherals through a Wishbone-style interface. This included handling access size, alignment, memory responses, and pipeline status during memory transactions.

Writeback, CSR, and Interrupt Handling

The writeback stage completes instruction execution by writing results back to the register file and handling CSR-related behavior. This part also involved trap/interrupt control, machine exception program counter behavior, and machine-mode control/status registers.

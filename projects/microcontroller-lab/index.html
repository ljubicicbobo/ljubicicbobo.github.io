Microcontroller Programming Lab

Hands-on embedded programming project using the Raspberry Pi Pico with the RP2040 microcontroller, focused on low-level peripheral control, ADC input, GPIO handling, I²C communication, PWM motor control, and hardware-based testing.

Course: Microcontroller, UE — TU Graz
Platform: Raspberry Pi Pico / RP2040
Language: C
Main topics: GPIO, ADC, PWM, I²C, motor control, debugging

Overview

This lab focused on practical microcontroller programming with the Raspberry Pi Pico and the RP2040 MCU. The course covered hands-on programming with a prebuilt hardware kit and included GPIO, ADC, PWM, and I²C communication tasks.

The hardware setup included a Raspberry Pi Pico, two switches, a brushless motor with encoder feedback, and a Raspberry Pi Debug Probe.

What I Implemented

  • Low-level GPIO configuration using RP2040 memory-mapped registers.
  • GPIO input/output handling, including pull-up and pull-down configuration.
  • ADC initialization and 12-bit analog value reading.
  • Translation of raw ADC values into discrete input sectors.
  • A finite-state-machine based vault application using switches, ADC input, and LED feedback.
  • Motor-control logic for a smart knob using PWM, I²C encoder feedback, snap points, and PID-style control.
  • Debugging and testing using serial output and the course hardware testsystem.

Technical Details

GPIO Driver

I implemented GPIO functions by directly accessing the RP2040 hardware register regions for IO_BANK0, PADS_BANK0, and SIO. The GPIO driver included pin function selection, input/output mode configuration, output level control, input reading, and pull resistor configuration.

ADC Driver

The ADC implementation initialized the ADC peripheral, configured the analog input pin, selected the ADC channel, started conversions, waited for the result, and returned the 12-bit ADC value. The ADC value was also translated into discrete sectors for use in the vault application.

Vault Application

The vault task used a finite state machine with states such as open, set PIN, enter PIN, wrong PIN, and locked. It used switch inputs, ADC-based rotary input, button confirmation, and LED feedback to simulate a small embedded access-control system.

Smart Knob Application

The smart knob task used encoder feedback, PWM motor control, I²C initialization, snap points, and PID-style control logic to create a motorized rotary interface. The goal was to create controlled movement toward defined angular positions instead of simply reading a passive input.

What I Learned

  • How microcontroller peripherals are configured below the abstraction level of normal libraries.
  • How GPIO direction, pin functions, pull resistors, and output levels are controlled through registers.
  • How ADC values can be sampled and converted into application-level input states.
  • How to structure embedded applications as state machines.
  • How PWM and encoder feedback can be combined for motor-control behavior.
  • How to debug embedded code using serial output, hardware testing, and a debug probe.
